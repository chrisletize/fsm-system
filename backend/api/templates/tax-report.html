üìö How to Generate Accurate Cash-Basis Tax Reports

Why This Matters: NC requires reporting tax based on PAYMENT DATE (cash basis), not invoice date. This means a June invoice paid in December counts toward December's tax liability.

1Ô∏è‚É£ Export Tax Report (All Invoices)

  • Go to ServiceFusion ‚Üí Reports ‚Üí Tax Report
  • Date Range: Select ENTIRE YEAR (or longer)
    • Example: 1/1/2025 - 12/31/2025
    • Do NOT filter to just current month
  • Why? You need ALL invoices, even old ones that might get paid this month
  • Export to Excel
  • Save as: TaxReport_[dates].xlsx

2Ô∏è‚É£ Export Transaction Report (Current Month Only)

  • Go to ServiceFusion ‚Üí Reports ‚Üí Customer Transaction Report
  • Date Range: Select ONLY the month you're filing
    • Example: 12/1/2025 - 12/31/2025
  • Why? You only want payments COLLECTED in this specific month
  • Export to Excel
  • Save as: TransactionReport_[dates].xlsx

3Ô∏è‚É£ Upload Both Files Here

  • Select your company from dropdown
  • Upload Tax Report (Step 1 file)
  • Upload Transaction Report (Step 2 file)
  • Click "Generate Report"

4Ô∏è‚É£ What The System Does

  • Matches invoices to payments by Job Number
  • Shows ONLY invoices that were paid in your selected month
  • Calculates state vs county tax breakdown
  • Result: Accurate cash-basis tax report for NC filing

üí° Example

Scenario:

  • June invoice: $1,000 + $70 tax
  • Customer pays in December

How it works:

  • Tax Report (full year): Has this invoice
  • Transaction Report (December): Has this payment
  • Result: $70 tax counts toward DECEMBER, not June

üîß Troubleshooting

  • "Numbers seem low" ‚Üí Tax Report might be filtered by date. Re-export with full year.
  • "Jobs missing" ‚Üí Make sure Transaction Report includes ALL payment types.
  • "Totals don't match" ‚Üí Verify both files are for the same company.
  • "Some invoices not showing" ‚Üí Those jobs haven't been paid yet in your selected month.
